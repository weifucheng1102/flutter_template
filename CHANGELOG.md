# Changelog

本文件记录模板版本的重要演进，重点描述会影响新项目初始化、模板维护和团队协作方式的改动。

## [2.0.0] - 2026-04-13

### Added

- 新增 `app/core` 基础设施层，补充应用启动封装、全局异常捕获、网络预热和存储封装。
- 新增 `app/routes` 路由体系，包括路由常量、路由表和登录守卫中间件。
- 新增启动分流页 `BootstrapPage`，统一处理协议确认、登录态和首页跳转。
- 新增登录页、工作台、设置页、受保护页面和协议文档页等模板骨架页面。
- 新增 `scripts/run_dev.sh`、`scripts/run_test_env.sh`、`scripts/run_prod.sh`，简化环境切换。
- 新增 `AGENTS.md` 和 `docs/` 文档体系，补充模板结构说明、启动链路说明、改造清单和模块开发规范。
- 新增 `photo_view` 直接依赖声明，补齐图片预览组件依赖关系。

### Changed

- 重构 `ServiceRequest`，升级为单例 `Dio` + 拦截器结构，统一请求发送、日志、错误处理和 token 注入。
- 将 iOS 首次数据网络访问授权场景从页面级硬编码实现，升级为可配置的网络预热服务，并支持首次失败后自动重试一次原请求。
- 将存储访问从散落的 `GetStorage` 读写，收敛为 `StorageKeys` 和 `StorageService`。
- 将应用初始化逻辑从 `main.dart` 抽离到 `AppBootstrap`，避免入口文件持续膨胀。
- 将启动页、导航页和协议页从 demo/临时实现重构为更适合模板复用的骨架实现。
- 收紧多个公共组件和工具类的类型定义，减少 `dynamic`、`Function?` 和模板遗留死代码。
- 重写 `README.md`，升级为模板使用手册而不是简单依赖清单。

### Fixed

- 修复图片网格组件依赖不存在资源图的问题，改为内置图标兜底。
- 修复图片预览组件缺少直接依赖声明的问题。
- 修复权限申请、公用弹窗和若干工具函数中的弱类型与上下文使用问题。
- 修复默认测试仍然指向计数器示例的问题，改为符合当前模板结构的基础 smoke test。
- 修复若干 Flutter 3.38 相关废弃 API 和静态分析警告。

### Removed

- 删除不应保留在模板仓库中的构建报告 `android/build/reports/problems/problems-report.html`。
- 删除一批无效占位代码、未使用导入和过时模板残留实现。

### Notes

- `networkWarmupUrl` 当前仍保留 `https://www.baidu.com` 作为兼容示例，建议在新项目中替换为自己的轻量健康检查接口。
- 新项目初始化后，建议优先修改 `AppConfig`、协议页、登录页和首页骨架。
